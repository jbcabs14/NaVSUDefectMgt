NaVSU v1.0 Functional [Campus Navigation App]: No 'NO results found' message when search does not match any landmark

[Summary]
 When a user inputs text that does not match with any name of available landmarks, the message 'results not found' does not show up for the user which leads to confusion for user whether it is due to interconnectivity issues or other reasons.

[Precondition]
 Software version: v1.0 Beta
 Software configuration: Reporting module enabled
 Hardware specifications: Huawei P30
 Network configuration: Connected to WiFi

[Steps to reproduce]
 1. Type "pool" in the search bar.
 2. Press the "Enter" key on the on-screen keyboard.

[Actual results]
 The keyboard closes, but no feedback (message or UI change) is shown.

[Expected results]
 A clear "No results found" message should appear on the screen.

 No, new implementation
